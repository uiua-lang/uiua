You can find the full changelog here.

You can download pre-built binaries here.

Uiua is a general purpose, stack-based, array-oriented programming language with a focus on tacit code.

Here are some highlights of this release:

Data Definitions

Data definitions are a newly stabilized feature that allows names to be given to the rows of heterogeneous arrays. This is similar to structures or classes in other languages.

There is new tutorial detailing all the features of data definitions, but here are a few examples:

Simple data definitions can be defined following a ~. We can then use the name of the definition to construct an array, and we can use the field names to access the fields.

~Customer {Name Phone Address} Customer "John Doe" "555-1234" "123 Main St" # {Name: "John Doe" Phone: "555-1234" Address: "123 Main St"} Customer~Name . # "John Doe" {Name: "John Doe" Phone: "555-1234" Address: "123 Main St"}

If we want to add functions associated with the data definition, we can put the definition inside a module. We can use a module import macro to easily access the items in the module.

Sided Subscripts

Sided subscripts are subscripts that instead of changing how a function works based on a number, do it based on a "side", either left or right.

Sided subscripts are formatted from __< and __>. They are currently only stabilized for РѕЕ both and РіЊ bracket . These cases simplify some common stack manipulation patterns.
